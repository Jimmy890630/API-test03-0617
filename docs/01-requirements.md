# 名片系統需求規格書

## 1. 專案概述

本專案為金屬加工產業開發的**名片管理系統**，採用原生 HTML、CSS、JavaScript 建置。系統核心目的為提供可視化名片頁面與可擴充的 API 串接結構。

> ⚠️ 開發原則：所有程式碼須待正式收到「開始開發」指令後才會生成。本文件僅做規劃與規格說明。

## 2. 名片欄位

| 欄位名稱 | 英文名稱 | 必填 | 說明 |
|---------|---------|------|------|
| 姓名 | `name` | 是 | 名片擁有者姓名 |
| 服務單位（公司名稱） | `company` | 是 | 所屬公司名稱 |
| 連絡電話 | `phone` | 是 | 行動或市話 |
| 電子信箱 | `email` | 是 | 聯絡信箱 |
| LINE ID | `lineId` | 否 | LINE 即時通訊 ID |
| 服務項目 | `services` | 是 | 金屬加工服務項目，可為多項 |
| 職稱 | `title` | 否 | 職位名稱 |
| 公司統編 | `taxId` | 否 | 台灣公司統一編號 |
| 傳真號碼 | `fax` | 否 | 公司傳真 |

## 3. 技術規範

- **前端技術**：原生 HTML5、CSS3（不使用框架）、原生 JavaScript（ES6+）
- **檔案拆分**：HTML / CSS / JS 分離，禁止混寫
- **程式碼品質**：所有程式碼段落須具備中文註解；JS 方法須註解使用方式、參數與回傳值
- **重複利用**：共用 JS 方法須集中於 `assets/js/utils.js`，禁止重複撰寫

## 4. 開發規則

1. 僅使用原生技術，不引入前端框架或 CSS 框架。
2. 建立基礎樣式檔案，樣式風格以簡約為主。
3. 建立 API 文件，待後端 API 確定後填入詳細規格。
4. 建立開發 log 文件，記錄開發里程碑與異動。
5. 建立檔案關係文件，說明各檔案職責與相依關係。
6. 未經詢問不得修改既有文件。
7. 所有規劃文件使用 Markdown 格式建立。

## 5. 命名規範（中文備註需求）

- HTML：`index.html`（入口頁）、`card.html`（名片頁，視需求）
- CSS：`assets/css/main.css`（主樣式）、`assets/css/card.css`（名片元件樣式）
- JS：`assets/js/main.js`（入口邏輯）、`assets/js/utils.js`（共用方法）
- 圖片：`assets/images/`
- 字型：`assets/fonts/`（若使用自訂字型）
